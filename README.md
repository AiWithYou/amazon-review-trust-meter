# Amazon レビュー注意度メーター

Amazon.co.jpの商品ページにある評価分布・レビュー本文・投稿日・購入確認・商品説明などをブラウザ内で分析し、購入前に追加確認したいポイントを表示するChrome拡張機能です。

> [!IMPORTANT]
> Amazon、Amazon.co.jp、Amazon Vine、サクラチェッカーの公式・提携拡張機能ではありません。表示する注意度は、レビュー不正の確率や個々の投稿者への断定ではありません。

## まずインストール

Chromeウェブストアでは配布していないため、ZIPを展開してChromeへ読み込みます。初回だけ「デベロッパー モード」を使用します。

### 1. 配布用ZIPをダウンロード

- [Amazon レビュー注意度メーター v2.1.0 配布用ZIPをダウンロード](https://github.com/AiWithYou/amazon-review-trust-meter/releases/download/v2.1.0/amazon-review-trust-meter-v2.1.0.zip)
- [SHA-256チェックサムを確認](https://github.com/AiWithYou/amazon-review-trust-meter/releases/download/v2.1.0/SHA256SUMS.txt)
- [v2.1.0のReleaseページを見る](https://github.com/AiWithYou/amazon-review-trust-meter/releases/tag/v2.1.0)

ZIPのままではChromeへ読み込めません。ダウンロード後、WindowsでZIPを右クリックして「すべて展開」を選びます。

展開後は、次のように `manifest.json` が入っているフォルダを使います。

```text
amazon-review-trust-meter-v2.1.0/
├── manifest.json
├── content.js
├── scoring.js
├── styles.css
└── ...
```

### 2. Chromeへ追加

1. Chromeのアドレスバーに `chrome://extensions` と入力して開きます。
2. 画面右上の「デベロッパー モード」をオンにします。
3. 画面左上の「パッケージ化されていない拡張機能を読み込む」を押します。
4. 展開した `amazon-review-trust-meter-v2.1.0` フォルダを選びます。
5. 拡張機能一覧に「Amazon レビュー注意度メーター」が表示されればインストール完了です。

フォルダを選ぶ画面では、`manifest.json` が直接入っているフォルダを選んでください。ZIPファイルや、その一つ外側のフォルダを選ぶと読み込めません。

### 3. Amazonの商品ページで使う

1. Amazon.co.jpの商品詳細ページを開きます。
2. すでに開いていたページは再読み込みします。
3. 商品名の近くにあるAmazonの評価欄の直下を確認します。

次のような商品詳細ページで動作します。

```text
https://www.amazon.co.jp/dp/商品のASIN
https://www.amazon.co.jp/商品名/dp/商品のASIN
```

Amazonのトップページ、検索結果、注文履歴などでは表示されません。

## 使い方

商品ページを開くと、評価欄の直下に次の4項目がコンパクトに表示されます。

| 表示 | 意味 |
|---|---|
| `★3.8` など | 表示レビューの偏りを保守的に調整した補正評価。参考値です |
| 判定 | 「目立つ異常は少ない」「確認推奨」「注意」「要注意」のいずれか |
| 注意度 `0〜100` | 購入前に追加確認したい度合い。高いほど確認材料が多い状態です |
| 詳細を見る | 判定根拠、分析材料、レビュー兆候、商品記載の注意点を開きます |

注意度の目安は次のとおりです。

| 注意度 | 表示 | 購入前の見方 |
|---:|---|---|
| 0〜24 | 目立つ異常は少ない | 表示範囲では強い注意材料が少ない状態です |
| 25〜49 | 確認推奨 | 低評価レビューや販売元などを追加確認してください |
| 50〜64 | 注意 | 複数の注意材料が重なっています |
| 65〜100 | 要注意 | 購入判断の前に判定根拠を詳しく確認してください |

注意度は「サクラである確率」ではありません。注意度が低くても安全を保証せず、分析材料が足りない場合は「判定材料不足」と表示します。

## 表示されないとき

上から順番に確認してください。

1. 開いているページがAmazon.co.jpの商品詳細ページか確認します。検索結果ページでは動きません。
2. 商品ページを再読み込みします。
3. `chrome://extensions` で「Amazon レビュー注意度メーター」がオンになっているか確認します。
4. 拡張機能カードの更新ボタンを押してから、商品ページを再読み込みします。
5. 読み込み時に選んだフォルダの直下に `manifest.json` があるか確認します。
6. 拡張機能カードにエラーが出ている場合は「エラー」を開いて内容を確認します。

Amazon側のページ構造変更や、商品ページの一部レイアウトでは、表示できない・表示位置がずれる場合があります。

## 更新・削除

### 新しいバージョンへ更新

1. 新しい配布用ZIPをダウンロードして展開します。
2. `chrome://extensions` を開きます。
3. 既存の「Amazon レビュー注意度メーター」を削除します。
4. 「パッケージ化されていない拡張機能を読み込む」から、新しいフォルダを選びます。
5. Amazonの商品ページを再読み込みします。

この拡張機能は設定やアカウント情報を保存しないため、削除して読み込み直しても引き継ぎ作業はありません。

### アンインストール

1. `chrome://extensions` で「Amazon レビュー注意度メーター」の「削除」を押します。
2. 展開した `amazon-review-trust-meter-v2.1.0` フォルダが不要なら、通常のファイルと同じように削除します。

## 判定の仕組み

AIや外部AI APIによる判定ではありません。ブラウザ内のルールベース処理で、複数の弱い兆候が同じ方向に重なっているかを確認します。

主に次の情報を使います。

- 星別評価の偏りと、Amazon表示平均との整合性
- レビュー本文の類似表現や短文・汎用表現
- 14日・30日・90日単位の投稿集中
- 星の方向とレビュー本文の内容が一致しているか
- 「Amazonで購入」、Vine、「参考になった」票
- 同じ商品ページ内にある同一投稿者の同方向レビュー
- 商品名と説明にある電池容量、防水等級、連続時間などの矛盾
- 強い宣伝表現や検証しにくい訴求が複数重なっているか

有名ブランドか無名ブランドかだけで安全・危険を決めることはありません。Vineレビュー、投稿日集中、レビュー数の多さ、ブランド名の表記差だけを理由に注意度を上げることもありません。

より詳しい仕様は[判定アルゴリズム](./docs/ALGORITHM.md)を参照してください。

## プライバシーと権限

- 分析はブラウザ内で完結します。
- 商品情報や閲覧履歴を外部AI API・開発者サーバーへ送信しません。
- `permissions` と `host_permissions` は要求しません。
- Amazon.co.jpの商品詳細ページだけで動作します。
- アカウント情報、Cookie、注文履歴、支払い情報を読む処理はありません。
- 「本家サクラチェッカーで確認」を押した場合だけ、対象商品のASINを含むページを新しいタブで開きます。

## 分析できない情報と限界

この拡張機能は、現在開いている商品ページから取得できる範囲だけを分析します。次の情報は取得しません。

- Amazon内部の購入・返品・返金・アカウント情報
- 削除済みレビューやレビュー件数の過去推移
- 商品ページに現在表示されていないレビュー
- 同じ投稿者による別商品のレビュー履歴
- メーカー・ブランド・販売者を横断した過去履歴
- 外部サイトの評判や検索結果

そのため、誤検知や見逃しはあり得ます。商品説明、販売元、低評価レビュー、返品条件なども併せて確認してください。

## 現在のバージョン

最新版は[v2.1.0](https://github.com/AiWithYou/amazon-review-trust-meter/releases/tag/v2.1.0)です。

- 発売直後の自然なレビュー集中を減衰し、複数窓走査による偶然の集中を抑制
- 小標本の比率判定を95% Wilson下限へ変更し、短文レビューの類似判定を厳格化
- 集中期間内外の平均星差と、同一投稿者の同方向レビューを複合兆候へ追加
- 高得点シグナルの境界を平滑化し、低件数の平均評価乖離を観測情報へ降格
- 参考票を上限付きの弱い減衰材料として利用し、信頼度がほぼ0のシグナル表示を廃止

### 必要環境

- Node.js 18以上
- PowerShell 7
- 外部npmパッケージへの依存なし

```powershell
npm test
npm run check
npm run package
npm run package:verify
```

- `npm run check`：判定ファイルと `content.js` の構文確認後、回帰テストを実行
- `npm run package`：Manifestのバージョンを使って配布ZIPと `SHA256SUMS.txt` を生成
- `npm run package:verify`：ZIPのファイル一覧・内容・Manifestバージョン・チェックサムを検証

## 免責事項

- 本ツールの表示は購入判断を補助する参考情報であり、レビュー不正を断定するものではありません。
- Amazon側のページ構造変更により、情報を取得できない、または表示位置が崩れる場合があります。
- 表示中のレビューは全レビューの無作為標本ではなく、注意度と補正評価には選択偏りがあります。
