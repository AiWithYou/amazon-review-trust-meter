# Amazon レビュー注意度メーター

Amazon.co.jpの商品詳細ページに表示されている商品情報、星別評価、レビュー本文、投稿日、購入確認、Vine表示を端末内で複合分析し、購入前に追加確認すべき度合いを表示するChrome拡張機能です。

Amazon、Amazon.co.jp、Amazon Vine、サクラチェッカーの公式・提携拡張機能ではありません。表示する注意度は、レビューが不正である確率や個々の投稿者への断定ではありません。

## v2.0で変わったこと

v1系の「怪しい特徴を単純加算し、注意度を★へ線形変換する」方式を廃止し、判定エンジンをDOM層から分離し、`scoring-base.js`、`scoring-features.js`、`scoring.js` の3層に整理しました。

- 本文の文字n-gram類似度から、単発の似た文章ではなくレビュー群を検出
- 14日・30日・90日の投稿集中を、取得期間内の期待件数と比較
- 高評価操作と低評価攻撃を対称に判定
- 本文重複、投稿集中、評価方向、購入確認、星分布が重なった場合に強く加点
- VineレビューとVineの発売時集中を、不正兆候として直接加点しない
- ブランド名が商品名にないことや、レビュー数が多いことだけでは加点しない
- データ取得量に応じた「判定確度」を別表示
- Amazon平均とは別に、疑わしい表示レビューの影響を小さく再重み付けした「補正評価（参考）」を表示
- 欠損データを安全材料として扱わず、材料不足なら判定不能にする

判定仕様の詳細は [`docs/ALGORITHM.md`](docs/ALGORITHM.md) を参照してください。

## 表示項目

- **注意度 0〜100**：複数の独立した異常兆候がどの程度重なったか
- **判定確度 0〜86%**：星分布、レビュー本文、投稿日などをどこまで取得できたか
- **補正評価（参考）**：表示レビューの疑わしさで星別分布を保守的に再重み付けした値
- **主な判定根拠**：点数、根拠、取得値
- **補足**：Vine集中、割合の丸め誤差、ブランド表記など、加点しない情報

判定の目安は次のとおりです。

| 注意度 | 表示 |
|---:|---|
| 0〜24 | 目立つ異常は少ない |
| 25〜49 | 確認推奨 |
| 50〜64 | 注意 |
| 65〜100 | 要注意 |

判定確度が不足する場合は、注意度が低くても「安全」とは表示せず、「判定材料不足」または低確度表示にします。

## 判定に使う情報

- 商品情報：過度に長い商品名、キーワード反復、強い宣伝表現、商品名と説明の仕様値不一致
- 評価分布：極端な★5集中、★5と★1の二極化、表示平均との不整合
- レビュー本文：短文・汎用表現、文字n-gramによる類似本文クラスタ、星と本文方向の不一致
- 投稿時系列：14日・30日・90日窓の投稿集中と、取得期間に対する超過度
- 投稿属性：Amazonで購入表示、Vine表示
- 複合兆候：投稿日集中と本文重複、評価方向、星分布が同時に一致するか
- 適用範囲：複数バリエーションのレビュー混在

## 判定に使わない情報

現在のローカル版では次の情報を取得しません。

- Amazon内部の購入、返品、返金、アカウント、アクセス情報
- 削除済みレビューとレビュー件数の過去推移
- 商品ページに現在表示されていないレビュー
- 同一レビュアーが別商品へ投稿した履歴
- メーカー、ブランド、販売者の横断的な過去履歴
- 外部サイトの評判や検索結果

この制約により、組織的なレビュー群を完全に識別することはできません。特に商品横断の共同行動グラフは、公開レビューを継続収集する別モードまたはバックエンドが必要です。

## プライバシーと権限

- 分析はブラウザ内で完結し、商品情報や閲覧履歴を外部API・開発者サーバーへ送信しません。
- `permissions` と `host_permissions` は要求しません。
- コンテンツスクリプトは、マニフェストに記載したAmazon.co.jpの商品詳細ページだけで動作します。
- 「本家サクラチェッカーで確認」をクリックした場合だけ、対象ASINを含むページを新しいタブで開きます。
- アカウント情報、Cookie、注文履歴、支払い情報を読む処理はありません。

## インストール

1. このリポジトリをダウンロードまたはクローンします。
2. Chromeで `chrome://extensions` を開きます。
3. 「デベロッパー モード」を有効にします。
4. 「パッケージ化されていない拡張機能を読み込む」から、このフォルダを選択します。
5. Amazon.co.jpの商品詳細ページを再読み込みします。

更新後は `chrome://extensions` で拡張機能の更新ボタンを押し、商品ページを再読み込みしてください。

## 開発・テスト

Node.js 18以上を使用します。外部パッケージへの依存はありません。

```bash
npm test
npm run check
```

`npm run check` は判定3ファイルと `content.js` の構文確認後、全テストを実行します。テストには正常な高評価商品、組織的な高評価操作、類似低評価攻撃、Vine発売時集中、投稿日集中のみ、欠損データ、日本語の言い換えテンプレートを含みます。

## ファイル構成

```text
manifest.json
scoring-base.js         正規化、分布、本文類似クラスタ。DOM非依存
scoring-features.js     時系列、レビュー単位リスク、確度、補正評価
scoring.js              特徴統合、点数、判定ラベル
content.js              Amazon DOM抽出、表示、再描画制御
styles.css              商品ページ内カードの表示
package.json
tests/scoring.test.cjs  パーサー、判定、誤検知抑制の回帰テスト
docs/ALGORITHM.md       判定仕様
```

## 免責事項

- Amazon側のページ構造変更により、情報を取得できない、または表示位置が崩れる場合があります。
- 表示中の少数レビューは全レビューの無作為標本ではないため、注意度と補正評価には選択偏りがあります。
- 誤検知や見逃しはあり得ます。商品説明、販売元、低評価レビュー、返品条件なども併せて確認してください。
